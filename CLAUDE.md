## Project Overview

A PHP-based AI chat web application with an admin management system. No build tools, no Node.js — pure PHP 8.0+ served by Apache/Nginx (XAMPP on Windows). All configuration is stored in MariaDB 10+, not in source files.

**Stack:** PHP 8.0+, MariaDB 10+, jQuery 3.7 (CDN), marked.js (CDN), highlight.js (CDN), Server-Sent Events for streaming.

## Running the Application

```bash
# Validate PHP syntax for any file
C:\xampp\php\php.exe -l <file.php>

# Run migrations (CLI)
C:\xampp\php\php.exe migrate.php

# Start XAMPP Apache + MariaDB from XAMPP Control Panel, then access:
# http://localhost/php-ai-chat-ui/install.php   ← first-time setup
# http://localhost/php-ai-chat-ui/migrate.php   ← run DB migrations
# http://localhost/php-ai-chat-ui/chat.php      ← end-user chat
# http://localhost/php-ai-chat-ui/admin.php     ← admin panel
```

## File Architecture

| File | Purpose |
|------|---------|
| `config.php` | DB constants only (`DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASS`). Auto-generated by `install.php`. Never hardcode credentials elsewhere. |
| `install.php` | One-shot setup wizard: creates DB, all tables, seeds default settings, writes `config.php`, creates first admin user. Should be deleted/renamed after use. |
| `migrate.php` | Migration runner. Add new schema changes here as versioned closures. Tracks applied migrations in `schema_migrations` table. Run after every schema change. |
| `admin.php` | Single-file admin panel. Handles auth (admin role required), CRUD for settings/users/models/conversations. Auto-runs `ensureModelsTable()` on every request as a safety migration. |
| `chat.php` | Single-file chat interface. Handles user login (username+password), SSE streaming endpoint (`?stream=1`), and three AJAX APIs (`?api=convs`, `?api=msgs`, `?api=del_conv`). |

## Database Schema

```
settings       (key PK, value, label, group)          ← API config, site name, etc.
users          (id, username UNIQUE, password bcrypt, display_name, role ENUM admin/user, is_active, last_login)
conversations  (id, user_id FK→users, title, model, created_at, updated_at)
messages       (id, conversation_id FK→conversations, role ENUM user/assistant/system, content LONGTEXT)
models         (id, name UNIQUE, label, is_active, sort_order)  ← AI model list for dropdown
schema_migrations (id PK, applied_at)                 ← tracks which migrations have run
```

Foreign keys use `ON DELETE CASCADE`. All tables: `ENGINE=InnoDB utf8mb4_unicode_ci`.

## Key Settings (stored in `settings` table)

| Key | Used by |
|-----|---------|
| `api_key` | chat.php → OpenAI/Ollama Authorization header |
| `base_url` | chat.php → endpoint base (e.g. `http://localhost:11434/v1`) |
| `model` | chat.php → default model on chat load |
| `system_prompt` | chat.php → prepended to every API call |
| `max_tokens` | chat.php → API `max_tokens` param |
| `site_name` | Both → page titles, login screen |

## Request Flow

### Chat user (`chat.php`)
1. No `config.php` → redirect `install.php`
2. DB fail → redirect `install.php`
3. Not logged in → show username/password login form (POST `chat_login=1`)
4. Logged in → serve chat HTML; JS loads conversation list via `?api=convs`
5. Send message → JS POSTs to `?stream=1` with `{messages, model, conv_id, conv_title}`
6. PHP creates/updates `conversations` row, saves user message, streams tokens via SSE, saves full assistant response after `curl_exec` completes
7. JS receives `conv_id` in first SSE frame to update sidebar

### Admin (`admin.php`)
1. No `config.php` → redirect `install.php`
2. `ensureModelsTable()` runs on every request (idempotent `CREATE TABLE IF NOT EXISTS`)
3. Not logged in (admin role required) → login form
4. All POST actions → process → `header('Location: ...')` redirect (PRG pattern)
5. Pages: `dashboard`, `settings`, `models`, `users`, `conversations`

## Adding a Database Migration

1. Open `migrate.php`
2. Add a new entry to `$migrations` array **at the end**:
```php
'20250201_001_your_description' => function(PDO $pdo): void {
    $pdo->exec("ALTER TABLE `users` ADD COLUMN `avatar` VARCHAR(500) NULL AFTER `display_name`");
},
```
3. Visit `http://localhost/php-ai-chat-ui/migrate.php` or run `php migrate.php`
4. Migration IDs must be unique and follow `YYYYMMDD_NNN_slug` format

## API / AI Provider Configuration

All set via **Admin → ตั้งค่า API**. No code changes needed to switch providers:

- **Ollama local:** `base_url = http://localhost:11434/v1`, `api_key = ollama`
- **OpenAI:** `base_url = https://api.openai.com/v1`, `api_key = sk-...`
- **OpenRouter:** `base_url = https://openrouter.ai/api/v1`, `api_key = sk-or-...`

The `models` table drives the model dropdown in chat. Manage via **Admin → จัดการ AI Models** (drag to reorder, toggle active/inactive).

## Auth System

- **Chat users:** any `users` row with `is_active=1`. Role doesn't matter for chat access.
- **Admin:** `users` row with `role='admin'` AND `is_active=1`. Session key: `admin_id`.
- **Chat session keys:** `chat_user_id`, `chat_username`, `chat_display_name`.
- Passwords: `password_hash(..., PASSWORD_BCRYPT, ['cost' => 12])` / `password_verify`.

## Streaming Implementation

`chat.php?stream=1` uses PHP + cURL with `CURLOPT_WRITEFUNCTION` callback to relay SSE tokens in real time. The full response is accumulated in `$fullResponse` and saved to the `messages` table **after** `curl_exec()` returns. The first SSE frame contains `{"conv_id": N}` so the JS can track the conversation ID for subsequent turns.

## Security Notes

- All DB queries use PDO prepared statements.
- HTML output uses `htmlspecialchars()` / the `e()` helper in `admin.php`.
- `config.php` must not be committed — add to `.gitignore` if using git.
- `install.php` should be deleted after first setup.
